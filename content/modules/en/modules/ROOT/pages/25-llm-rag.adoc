== Overview


NOTE: **Workshop pacing (~4 hours hands-on)** — Each module includes **Overview-only** steps (demo/visualization, ~5–10 min) and **Hands-on** steps (full lab). Skip Hands-on blocks to run a ~90-minute executive tour.

Large language models and retrieval-augmented generation (RAG) on OpenShift combine centralized inference with domain-specific context from factory docs, runbooks, and sensor logs — without exporting proprietary data to public SaaS. Red Hat Developer Hub Lightspeed assists developers in-catalog; Kairos Console agents answer scaling questions; Continue AI in DevSpaces suggests code inline using the same MaaS backend.

RAG architecture in hybrid manufacturing typically indexes PDFs and SOPs into a vector store (customer choice) while the LLM runs on link:https://docs.redhat.com/en/documentation/red_hat_openshift_ai_self-managed[OpenShift AI] ModelMesh or external MaaS. In this lab, you configure Lightspeed deployment in `charts/all/developer-hub/templates/lightspeed.yaml` and test prompts against the shared MaaS endpoint — observe latency and token usage suitable for shop-floor Wi-Fi constraints.

As `%USER_NAME%`, open Developer Hub, trigger Lightspeed on a catalog Component, and compare responses with Continue AI in a DevSpaces workspace. Module 27 extends the same MaaS path to multimodal NeuroFace chat — proving one governance model serves dev, ops, and end-user AI surfaces.

*Lightspeed sidecar deployment* — Llama Stack + LCS containers in the Developer Hub pod:

[source,yaml]
----
# charts/all/developer-hub/templates/lightspeed.yaml (excerpt)
apiVersion: backstage.io/v1alpha1
kind: Backstage
metadata:
  name: developer-hub
spec:
  application:
    extraContainers:
      - name: llama-stack
        image: quay.io/rh-lightspeed/llama-stack:latest
        env:
          - name: LLAMA_STACK_CONFIG
            value: /etc/lightspeed/llama-stack-run.yaml
          - name: OPENAI_API_KEY
            valueFrom:
              secretKeyRef:
                name: developer-hub-oidc-auth
                key: LIGHTSPEED_API_KEY
        volumeMounts:
          - name: lightspeed-config
            mountPath: /etc/lightspeed
      - name: lcs
        image: quay.io/rh-lightspeed/lcs:latest
        args: ["--model", "granite-3.1-8b-instruct"]
----

*RAG index pattern* — customer choice for vector store; LLM stays on OpenShift AI / MaaS:

[source,yaml]
----
# Example notebook cell — embed factory SOPs (run in ai-%USER_NAME% workshop-notebook)
# 1. Chunk TechDocs / PDF runbooks from Developer Hub entity repos
# 2. Store embeddings in customer vector DB (pgvector, Milvus, OpenSearch)
# 3. Retrieve top-k chunks → inject into MaaS chat/completions prompt
rag_query:
  question: "What is the safe shutdown procedure for machine-sensor-1?"
  context_docs: 5                    # top-k retrieved chunks
  model: llama-scout-17b
  endpoint: https://ai-gateway.%HUB_DOMAIN%/v1/chat/completions
  auth: APIKEY                       # Kuadrant key from module 23
----

== Features, benefits & cloud configuration

=== Key features

* **Lightspeed** in Developer Hub for catalog-aware prompts.
* **RAG** pattern: vector store + OpenShift AI inference (lab uses MaaS endpoint).
* **Continue AI** in DevSpaces for inline code suggestions.

=== Business benefits

* Factory runbooks and SOPs ground LLM answers — fewer hallucinations on the shop floor.
* Same MaaS backend as NeuroFace and AI Gateway — unified governance.
