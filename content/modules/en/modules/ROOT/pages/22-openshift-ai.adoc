== Overview


NOTE: **Workshop pacing (~4 hours hands-on)** — Each module includes **Overview-only** steps (demo/visualization, ~5–10 min) and **Hands-on** steps (full lab). Skip Hands-on blocks to run a ~90-minute executive tour.

OpenShift AI on the hub runs **ModelMesh + Serverless (Knative)** via `default-dsc`. Each user owns project **`ai-%USER_NAME%`** with pre-provisioned **Jupyter notebook** `workshop-notebook`, MaaS secret, and Developer Hub catalog entity. The configuration lives in `charts/all/openshift-ai-hub/`.

*App-of-Apps valuesObject for OpenShift AI* configures model serving modes, user projects, MCP integration, and available MaaS models — all declarative:

[source,yaml]
----
# charts/region/hub/templates/component-applications.yaml — openshift-ai-hub valuesObject
helm:
  valuesObject:
    clusterDomain: {{ $domain }}
    userCount: {{ $.Values.userCount | default 50 }}
    userProjects:
      enabled: true               # creates ai-user1..ai-userN projects
      notebook:
        enabled: true             # pre-provisions workshop-notebook per user
    dashboardExtensions:
      enabled: true
    modelServing:
      modelMeshEnabled: true      # shared multi-model serving
      serverlessEnabled: true     # KServe for dedicated single-model
      defaultDeploymentMode: ModelMesh
    mcp:
      enabled: true
      deployServer: true          # deploys ods-maas-mcp-server in maas-workshop
    maas:
      endpoint: {{ $.Values.litemaas.apiUrl }}
      apiKey: {{ $.Values.litemaas.apiKey }}
      models:
        - id: llama-scout-17b
          displayName: Llama Scout 17B (workshop default)
        - id: deepseek-r1-distill-qwen-14b
          displayName: DeepSeek R1 Distill Qwen 14B
        - id: codellama-7b-instruct
          displayName: CodeLlama 7B Instruct
----

*DataScienceCluster* — the operator reconciles all RHOAI components from one CR:

[source,yaml]
----
# charts/all/openshift-ai-hub/templates/datasciencecluster.yaml
apiVersion: datasciencecluster.opendatahub.io/v1
kind: DataScienceCluster
metadata:
  name: default-dsc
spec:
  components:
    dashboard:
      managementState: Managed          # ODS dashboard UI
    workbenches:
      managementState: Managed          # Jupyter / VS Code workbenches
    modelmeshserving:
      managementState: Managed          # shared multi-model runtime
    datasciencepipelines:
      managementState: Managed          # Kubeflow-style pipelines
    kserve:
      managementState: Managed          # Knative-based single-model serving
      serving:
        managementState: Managed
        defaultDeploymentMode: ModelMesh  # workshop default; MaaS proxies use RawDeployment
----

*InferenceService `workshop-sklearn`* — one sklearn model per user project via ModelMesh:

[source,yaml]
----
# charts/all/openshift-ai-hub/templates/user-projects.yaml (per ai-userN)
apiVersion: serving.kserve.io/v1beta1
kind: InferenceService
metadata:
  name: workshop-sklearn
  namespace: ai-%USER_NAME%
  annotations:
    serving.kserve.io/deploymentMode: ModelMesh   # shared runtime, not dedicated pod
spec:
  predictor:
    model:
      modelFormat:
        name: sklearn
      storage:
        key: minio
        path: anomaly-detection/model    # artifact in hub MinIO data lake
      resources:
        requests:
          cpu: "100m"
          memory: 512Mi
----

*MaaS proxy InferenceService* — external LLM routed through OpenShift AI (RawDeployment):

[source,yaml]
----
# charts/all/openshift-ai-hub/templates/maas-proxies.yaml
apiVersion: serving.kserve.io/v1beta1
kind: InferenceService
metadata:
  name: maas-llama-scout-17b
  namespace: maas-workshop
  annotations:
    serving.kserve.io/deploymentMode: RawDeployment  # dedicated pod for external API proxy
spec:
  predictor:
    containers:
      - name: kserve-container
        image: quay.io/opendatahub/openvino_model_server:latest
        env:
          - name: OPENAI_API_BASE
            valueFrom:
              secretKeyRef:
                name: openshift-ai-maas-credentials
                key: OPENAI_API_BASE     # https://maas-rhdp.apps.maas.redhatworkshops.io/v1
          - name: OPENAI_API_KEY
            valueFrom:
              secretKeyRef:
                name: openshift-ai-maas-credentials
                key: OPENAI_API_KEY
        args:
          - --rest_port=8080
          - --model_name=llama-scout-17b
----

[source,bash]
----
# Confirm DataScienceCluster is Ready

# Check your user project and notebook

# List available MaaS models
oc get configmap ods-mcp-server-registration -n maas-workshop -o yaml

# MCP server deployment
oc get pods -n maas-workshop -l app=ods-maas-mcp-server
----

== Features, benefits & cloud configuration

=== Key features

* **DataScienceCluster** (`default-dsc`) — ModelMesh, KServe, dashboard, notebooks.
* Per-user project **`ai-%USER_NAME%`** with `workshop-notebook` and InferenceService `workshop-sklearn`.
* MaaS playground in `maas-workshop` — shared LLM for all AI modules.

=== Business benefits

* Data scientists and developers share one governed inference layer.
* Notebooks and pipelines stay on-cluster — no export of factory data to public SaaS.

=== AWS — SageMaker vs OpenShift AI

[source,bash]
----
# SageMaker endpoint (cloud alternative — lab uses OpenShift AI instead)
aws sagemaker create-model --model-name sklearn-factory --primary-container   Image=246618743249.dkr.ecr.us-east-1.amazonaws.com/sagemaker-scikit-learn:1.2-1-cpu-py3,ModelDataUrl=s3://models/sklearn.tar.gz
aws sagemaker create-endpoint-config --endpoint-config-name sklearn-cfg   --production-variants VariantName=AllTraffic,ModelName=sklearn-factory,InitialInstanceCount=1,InstanceType=ml.m5.large
aws sagemaker create-endpoint --endpoint-name sklearn-factory --endpoint-config-name sklearn-cfg
----
