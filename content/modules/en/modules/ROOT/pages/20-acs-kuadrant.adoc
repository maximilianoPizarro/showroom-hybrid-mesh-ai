== Overview

NOTE: **Workshop pacing (~4 hours hands-on)** — Each module includes **Overview-only** steps (demo/visualization, ~5–10 min) and **Hands-on** steps (full lab). Skip Hands-on blocks to run a ~90-minute executive tour.

**Red Hat Advanced Cluster Security (ACS)** scans images and enforces runtime policies across ACM-managed clusters. SecuredCluster agents on spokes report to ACS Central on the hub (`charts/all/acs-init-bundle-sync/`).

**Red Hat Connectivity Link (RHCL)** includes **Kuadrant** controllers for Gateway API ingress and API management — not a separate product beside RHCL. Chart `charts/all/workshop-kuadrant-apis/` deploys **two dedicated gateways**:

[cols="2,2,3", options="header"]
|===
| Gateway | Namespace | Routes (path prefix)

| `workshop-apis.%HUB_DOMAIN%`
| `workshop-kuadrant-apis`
| `/httpbin`, `/countries`, `/mcp`

| `ai-gateway.%HUB_DOMAIN%`
| `ai-gateway-system`
| `/v1` → MaaS LLM (xref:23-ai-gateway.adoc[module 23])
|===

Each route binds **AuthPolicy**, **PlanPolicy**, and (for LLM) **TokenRateLimitPolicy** to its `HTTPRoute`. **APIProduct** CRs (`devportal.kuadrant.io/v1alpha1`) publish OpenAPI/Swagger links into Developer Hub.

=== APIProduct and plans (workshop-apis)

[source,yaml]
----
# charts/all/workshop-kuadrant-apis/templates/apiproducts.yaml
apiVersion: devportal.kuadrant.io/v1alpha1
kind: APIProduct
metadata:
  name: workshop-httpbin
  namespace: workshop-kuadrant-apis
spec:
  displayName: Workshop httpbin (httpbin.org)
  publishStatus: Published
  approvalMode: automatic              # key request auto-approved in lab
  documentation:
    openAPISpecURL: https://httpbin.org/spec.json
  targetRef:
    group: gateway.networking.k8s.io
    kind: HTTPRoute
    name: workshop-httpbin
----

Plan tiers on workshop-apis routes: **bronze** (5 req/min), **silver** (15), **gold** (60) — see `charts/all/workshop-kuadrant-apis/values.yaml`.

== Hands-on — terminal and curl

Use the embedded Showroom terminal (hub). Works for `%USER_NAME%` and `platformadmin`.

. Request key in Developer Hub (step 1a or 1b above).
. In the terminal:

[source,bash]
----
# Read-only oc via Showroom ServiceAccount (HTTPRoutes, APIProducts, policies)
sa-login
kuadrant-verify-oc

# Or as workshop user (same lab-reader permissions after hub-login)
hub-login user1
kuadrant-verify-oc

# Quick curl helpers
kuadrant-set-key "<paste-kuadrant-api-key>"
kuadrant-test-httpbin

# Or manual curl:
curl -sk -o /dev/null -w 'httpbin without key: %{http_code}\n' \
  https://workshop-apis.%HUB_DOMAIN%/httpbin/get

export KEY="$KUADRANT_API_KEY"

curl -sk -H "Authorization: APIKEY $KEY" \
  https://workshop-apis.%HUB_DOMAIN%/httpbin/get

curl -sk -H "Authorization: APIKEY $KEY" \
  https://workshop-apis.%HUB_DOMAIN%/countries/name/chile | head -c 400

oc get httproute -n workshop-kuadrant-apis
oc get apiproducts -n workshop-kuadrant-apis
----

TIP: Wrong header format (`Bearer` instead of `APIKEY`) returns **401** even with a valid key.

== Features, benefits & cloud configuration

=== Key features

* **ACS Central** — vulnerability and runtime policy for containers across fleet.
* **Connectivity Link (RHCL)** — Kuadrant APIProduct, AuthPolicy, PlanPolicy on dedicated gateways.
* Developer Hub Kuadrant plugin — self-service key minting + Swagger for published APIs.

=== Business benefits

* Security and API governance on one OpenShift footprint — no separate API gateway appliance.
* `%USER_NAME%` keys isolate usage for chargeback and audit.

=== AWS — WAF + Secrets Manager

[source,bash]
----
aws secretsmanager create-secret --name kuadrant-api-keys --secret-string '{"user1":"..."}'
aws wafv2 associate-web-acl --web-acl-arn arn:aws:wafv2:... --resource-arn arn:aws:elasticloadbalancing:...
----

=== Azure — Key Vault + Front Door

[source,bash]
----
az keyvault secret set --vault-name hybrid-kv --name kuadrant-user1 --value <api-key>
az network front-door waf-policy create --resource-group rg-workshop --name api-waf --sku Premium_AzureFrontDoor
----
