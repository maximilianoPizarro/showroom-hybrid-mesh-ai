= NeuroFace — Computer Vision Journey

++++
<span class="workshop-time-badge">~30 min</span>
++++

image::27-neuroface.png[NeuroFace — PPE detection CV journey with KServe and Skupper,960]

NeuroFace is a multimodal AI demo that combines **YOLO PPE detection** (hardhat, safety-vest, goggles), **face recognition** (OpenCV/OpenVINO), **object detection** (80 COCO classes), and **MaaS chat** (LLM) — deployed as a KServe InferenceService on Red Hat OpenShift AI with a federated east/west computer vision journey via Skupper.

== Architecture

image::27-neuroface-architecture.png[NeuroFace System Architecture — Angular frontend + FastAPI backend + YOLO PPE + OVMS + Chat,960]

.NeuroFace v1.5.0 architecture
[cols="1,1", options="header"]
|===
| Capability | Backend

| PPE detection (hardhat, safety-vest, goggles)
| KServe **InferenceService** `yolo-ppe-serving` (pre-built `neuroface-ppe-serving:v1.5.0`, KServe v1+v2)

| Face / webcam detection
| OVMS **`face-detection-retail-0005`** (optional, `ovms.enabled: true`)

| Object detection (80 COCO classes)
| YOLOv4-tiny (OpenCV DNN, bundled in backend image)

| Operator chat
| MaaS `/v1/chat/completions` (`llama-scout-17b`) — direct or via AI Gateway

| PPE events
| Kafka `cv.ppe.detections` → Camel K → Mailpit alerts

| PPE Workbench
| OpenShift AI **`ppe-workbench`** notebook (`ppe-detection.ipynb`)

| PPE Retrain Workbench
| OpenShift AI **`ppe-retrain-workbench`** notebook (MinIO S3 data connection)

| Model storage
| MinIO `s3://models/ppe-detection/model/best.pt` (hub → Skupper → spokes)

| Deployment
| `charts/all/neuroface/` on hub

| URLs
| `https://neuroface.%HUB_DOMAIN%` · `https://ppe-workbench.%HUB_DOMAIN%`
|===

== Computer Vision Journey (hub-and-spoke)

The **NeuroFace CV** journey federates PPE inference across east and west spokes — the same YOLO model deployed on two edge clusters with 50/50 traffic distribution via the hub gateway.

image::27-neuroface-ppe-sequence.png[PPE Detection Sequence — Browser → Frontend → Backend → YOLO → detections,960]

.CV journey components
[cols="1,2", options="header"]
|===
| Component | Description

| **Hub gateway** (`neuroface-gateway`)
| Gateway API with weighted HTTPRoute (50% east / 50% west)

| **Spoke inference** (`spoke-neuroface-cv`)
| KServe InferenceService `yolo-ppe-serving` in `neuroface-cv` namespace (HPA min 1, max 4)

| **Skupper**
| Connectors on each spoke publish `yolo-ppe-serving:8080`; hub listeners bridge into gateway

| **Ambient mesh**
| `neuroface-cv` namespace has `istio.io/dataplane-mode: ambient` (L4 mTLS + telemetry)

| **Grafana dashboard**
| `neuroface-cv` — gateway req/s, east vs west split, inference latency

| **Public Route**
| `https://neuroface-cv.%HUB_DOMAIN%` (inference-only: `/health`, `/v1/predict`, `/v2/models/yolo-ppe/infer`)
|===

== Helm values

[source,yaml]
----
# charts/all/neuroface/values.yaml (v1.5.0)
neuroface:
  ppe:
    enabled: true
    endpoint: http://yolo-ppe-serving:8080
    classes: hardhat,safety-vest,goggles
    confidence: "0.15"
    serving:
      image:
        repository: quay.io/maximilianopizarro/neuroface-ppe-serving
        tag: v1.5.0
    kafka:
      enabled: true
      bootstrap: "kafka-east-tst.service-interconnect.svc:9092"
      topic: "cv.ppe.detections"
      camelConsumer:
        enabled: true
    dataPersistence:
      enabled: true
      s3Endpoint: http://minio.industrial-edge-ml-workspace.svc:9000
      s3Bucket: models
      s3Prefix: ppe-detection/training-data
  chat:
    enabled: true
    # Direct MaaS (default). For Kuadrant API keys use AI Gateway:
    # modelEndpoint: "https://ai-gateway.<hub-domain>/v1"
    modelEndpoint: https://maas-rhdp.apps.maas.redhatworkshops.io/v1
    modelName: llama-scout-17b

yoloPpeServing:
  enabled: true
  image: quay.io/maximilianopizarro/neuroface-ppe-serving:v1.5.0
  # KServe InferenceService + ServingRuntime (RawDeployment)
  # Model downloaded from MinIO s3://models/ppe-detection/model/best.pt
----

== Hands-on: PPE Safety (webcam)

. Open link:https://neuroface.%HUB_DOMAIN%[NeuroFace] in your browser
. Click **PPE Safety** tab
. Allow webcam access — the YOLO model detects persons, hardhats, safety vests, and goggles in real-time
. The status bar should show: `PPE ENABLED` · `REACHABLE` · classes detected
. Try the **AI Chat** tab — ask _"What PPE is this worker wearing?"_

== Hands-on: PPE Workbench (image prediction)

. **Start the workbench** — OpenShift Console → **AI** → **Workbenches** → namespace `neuroface` → **ppe-workbench** → **Start**
. **Open the notebook** — link:https://ppe-workbench.%HUB_DOMAIN%[ppe-workbench.%HUB_DOMAIN%]
. **Run cells in order:**
.. Setup — verifies `PPE_ENDPOINT` health and NeuroFace `/api/ppe/status`
.. Upload — place a `.jpg` in `uploads/sample.jpg`
.. Detect — calls `POST /v1/predict` on `yolo-ppe-serving`
.. Visualize — bounding boxes for hardhat, safety-vest, goggles
. **Expected output** — JSON detections with `class_name`, `confidence`, and `bbox`; matplotlib overlay
