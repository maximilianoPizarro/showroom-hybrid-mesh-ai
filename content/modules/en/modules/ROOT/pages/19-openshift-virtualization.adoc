== Overview

**OpenShift Virtualization** (KubeVirt) runs VMs on the same OpenShift cluster as containers â€” shared RBAC, networking, and GitOps. Factory teams lift-and-shift legacy SCADA HMIs or Windows tooling without a separate VMware estate.

This workshop ships:

* Hub subscription `kubevirt-hyperconverged` (clustergroup app `cnv-example`)
* Cirros-based **`workshop-cnv-demo`** with cloud-init user `cirros`
* Developer Hub template publishing VM YAML to your Gitea org

Workshop IdP users get **view** on `cnv-workshop` via `platform-users` RoleBindings.

== Hands-on

=== Verify CNV operator

[source,bash]
----
hub-login
oc get csv -n openshift-cnv | grep -E 'NAME|kubevirt'
oc get kubevirt,kubevirt.kubevirt.io -n openshift-cnv 2>/dev/null || oc get hyperconverged -n openshift-cnv
oc get vm,vmi -n cnv-workshop
----

Expected: HyperConverged **Available**; VM **Running** or **Starting** when nested virt is enabled.

=== Console login to demo VM (optional)

[source,bash]
----
# Requires virtctl plugin in terminal image; otherwise use Console â†’ VirtualMachines â†’ Console
virtctl console workshop-cnv-demo -n cnv-workshop
----

Cloud-init default user: **cirros** (passwordless console on Cirros image).
