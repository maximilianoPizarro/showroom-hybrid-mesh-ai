== Overview


NOTE: **Workshop pacing (~4 hours hands-on)** — Each module includes **Overview-only** steps (demo/visualization, ~5–10 min) and **Hands-on** steps (full lab). Skip Hands-on blocks to run a ~90-minute executive tour.

NeuroFace combines **OVMS local vision** (face/object detection via `charts/all/neuroface/` with `ovms.enabled: true`) and **MaaS chat** at `/api/chat`. Hub **ModelMesh** serves platform models; NeuroFace OVMS handles low-latency webcam inference on the app namespace. See link:https://docs.redhat.com/en/documentation/red_hat_openshift_ai_self-managed[OpenShift AI documentation] for model serving patterns.

**Overview-only (~10 min):** Developer Hub → **neuroface-workshop** → Topology; open UI without webcam.

**Hands-on (~30 min):** Register as `%USER_NAME%`, test webcam + chat, inspect OVMS Service/route in namespace `neuroface`.

Catalog: System `hybrid-mesh-ai-platform` → Component **neuroface-workshop** (links UI + OpenShift AI dashboard).

*Helm values* — NeuroFace chart configuration on hub:

[source,yaml]
----
# charts/all/neuroface/values.yaml
replicaCount: 1
route:
  enabled: true
  hostname: neuroface.%HUB_DOMAIN%
ovms:
  enabled: true                        # local webcam inference — low latency
  modelName: face-detection-0200
  modelPath: /models/face-detection
maas:
  enabled: true
  apiBase: https://maas-rhdp.apps.maas.redhatworkshops.io/v1
  model: llama-scout-17b
  secretName: neuroface-maas-credentials  # from External Secrets, never Git
chat:
  endpoint: /api/chat                    # OpenAI-compatible proxy to MaaS
----

*OVMS InferenceService* — face detection model served locally in `neuroface` namespace:

[source,yaml]
----
apiVersion: serving.kserve.io/v1beta1
kind: InferenceService
metadata:
  name: neuroface-ovms
  namespace: neuroface
  annotations:
    serving.kserve.io/deploymentMode: RawDeployment
spec:
  predictor:
    model:
      modelFormat:
        name: openvino
      runtime: ovms-1.x
      storage:
        key: minio
        path: neuroface/face-detection-0200
      resources:
        requests:
          cpu: "500m"
          memory: 1Gi
----

== Features, benefits & cloud configuration

=== Key features

* **OVMS** local vision inference for webcam latency-sensitive detection.
* **MaaS chat** at `/api/chat` for operator Q&A.
* Catalog entity **neuroface-workshop** links UI, routes, and OpenShift AI.

=== Business benefits

* Vision at edge; generative answers from governed hub MaaS — split latency/cost optimally.
* Demo of multimodal AI in factory safety and training scenarios.
