== Welcome

Welcome to the **Hybrid Mesh AI Workshop** — a dual-track experience that mirrors how Red Hat customers run hybrid cloud on OpenShift. Part A (modules 01–05) is executive: strategy, ROSA architecture, security at scale, AWS integration patterns, and real industry outcomes. Part B (modules 10–28) is hands-on on a live RHDP hub-spoke fleet with ACM, GitOps, Industrial Edge, OpenShift AI, and NeuroFace.

== Red Hat product catalog in this workshop

=== OpenShift Container Platform
* **Advanced Cluster Management (ACM)** — fleet governance, policies, and GitOps-driven placement across hub and spokes.
* **OpenShift GitOps** — Argo CD Applications and ApplicationSets sync workshop components from Git.
* **Red Hat Advanced Cluster Security (ACS)** — runtime threat detection and compliance without ambient mesh in `stackrox`.
* **Kairos** — SmartScalingPolicy recommendations for Industrial Edge sensor workloads.
OpenShift is the consistent control plane whether you run on-prem, edge, or ROSA in AWS.

=== Red Hat Application Foundation
* **OpenShift Service Mesh (OSSM 3)** — ambient mesh with ztunnel, Kiali, and distributed tracing for IE apps.
* **Apache Camel on Kubernetes** — integrations and CDC pipelines in the Industrial Edge demo.
* **Connectivity Link / Kuadrant** — APIProduct, AuthPolicy, and rate limiting at the hub gateway.
Application Foundation delivers connectivity, integration, and API management as first-class OpenShift operators.

=== Red Hat Advanced Developer Suite
* **Developer Hub (Backstage)** — software catalog, scaffolder templates, and Topology for multicluster views.
* **Software Templates** — golden paths for Industrial Edge, Camel, CNV, and OpenShift AI workspaces.
* **Red Hat Dev Spaces** — cloud IDE for Kaoto, Continue AI, and template-driven development.
Developer Hub turns OpenShift into an internal developer platform with guardrails baked into every template.

=== OpenShift AI
* **DataScienceCluster (DSC)** — unified operator for notebooks, serving, and model mesh on the hub.
* **Model-as-a-Service (MaaS)** — shared LLM endpoint consumed by NeuroFace, Lightspeed, and DevSpaces.
* **NeuroFace** — webcam object/face detection plus chat backed by MaaS (not LibreChat).
OpenShift AI keeps models, pipelines, and inference on the same Kubernetes substrate as your applications.

=== OpenShift Virtualization
* **Container-native virtualization (CNV)** — VM workloads alongside containers via the workshop CNV demo template.
CNV lets you lift-and-shift legacy factory systems while new microservices run in the same OpenShift project.

=== Community & third-party
* **Gitea** — per-user org `ws-pass:[%USER_NAME%]` stores scaffolded Git repos for Argo CD.
* **MinIO** — object storage for Kubecost federated ETL and model artifacts.
* **Kubecost** — FinOps allocations across hub primary and spoke agents.
* **Quay** — container registry referenced by GitOps and template outputs.
These components integrate through OpenShift operators and GitOps, matching how customers compose best-of-breed stacks on OCP.

== Overview

This workshop demonstrates how Red Hat customers unify strategy and operations across hybrid cloud using OpenShift as the common platform. You will see the same patterns used with ROSA on AWS applied to a live RHDP hub-spoke lab with east and west spokes managed by ACM.

Part A frames the business case: modernization, security, FinOps, and AI without vendor lock-in. Part B lets you deploy Industrial Edge workloads, observe them multicluster, secure APIs with Kuadrant, and run inference with OpenShift AI and NeuroFace — all as `pass:[%USER_NAME%]` in an isolated namespace slice.

Use the hybrid callout table on this page whenever a facilitator compares production ROSA topology to what you see in the console. Registration at `https://workshop-registration.{hub_domain}` is required before hands-on modules.
