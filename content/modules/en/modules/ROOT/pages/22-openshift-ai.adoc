== Overview

NOTE: **Workshop pacing (~4 hours hands-on)** — Each module includes **Overview-only** steps (demo/visualization, ~5–10 min) and **Hands-on** steps (full lab). Skip Hands-on blocks to run a ~90-minute executive tour.

OpenShift AI on the hub runs **dashboard + workbenches** via `default-dsc`. Each user owns project **`ai-%USER_NAME%`** with pre-provisioned workbench **`neuroface-ml-lab`**, MaaS secret `maas-connection`, and Developer Hub catalog entity. Shared LLM access lives in **`maas-workshop`** (playground + MaaS InferenceService proxies). Configuration: `charts/all/openshift-ai-hub/`.

*Chart values (workshop hub track)* — RawDeployment for MaaS proxies; per-user projects from `userProjects`:

[source,yaml]
----
# charts/all/openshift-ai-hub/values.yaml (excerpt)
userCount: 50
userProjects:
  enabled: true
  notebook:
    enabled: true
modelServing:
  modelMeshEnabled: false      # workshop hub: simpler RawDeployment MaaS proxies
  serverlessEnabled: false
  defaultDeploymentMode: RawDeployment
maas:
  endpoint: https://maas-rhdp.apps.maas.redhatworkshops.io
  models:
    - id: llama-scout-17b
    - id: granite-3-2-8b-instruct
    - id: deepseek-r1-distill-qwen-14b
----

*Per-user namespace* — GitOps creates `ai-userN`, RBAC, MaaS secret, and Notebook:

[source,yaml]
----
# charts/all/openshift-ai-hub/templates/user-projects.yaml (per ai-userN)
apiVersion: kubeflow.org/v1
kind: Notebook
metadata:
  name: neuroface-ml-lab
  namespace: ai-%USER_NAME%
spec:
  template:
    spec:
      containers:
        - name: neuroface-ml-lab
          env:
            - name: OPENAI_API_BASE
              value: https://maas-rhdp.apps.maas.redhatworkshops.io/v1
            - name: OPENAI_API_KEY
              valueFrom:
                secretKeyRef:
                  name: maas-connection
                  key: api-key
----

*MaaS proxy InferenceService* — external LLM routed through OpenShift AI (RawDeployment in `maas-workshop`):

[source,yaml]
----
# charts/all/openshift-ai-hub/templates/maas-proxies.yaml
apiVersion: serving.kserve.io/v1beta1
kind: InferenceService
metadata:
  name: maas-llama-scout-17b
  namespace: maas-workshop
  annotations:
    serving.kserve.io/deploymentMode: RawDeployment
spec:
  predictor:
    containers:
      - name: kserve-container
        env:
          - name: OPENAI_API_BASE
            valueFrom:
              secretKeyRef:
                name: openshift-ai-maas-credentials
                key: OPENAI_API_BASE
          - name: OPENAI_API_KEY
            valueFrom:
              secretKeyRef:
                name: openshift-ai-maas-credentials
                key: api-key
----

**Overview-only (~10 min):** Catalog → **OpenShift AI — %USER_NAME%** → open dashboard; show Playground in `maas-workshop` (do not run notebook).

**Hands-on (~30 min):** Launch **`neuroface-ml-lab`**, run the embedded MaaS cell, open **NeuroFace** for OVMS PPE Safety webcam demo. In **maas-workshop** → **Models**, confirm MaaS proxy InferenceServices are Ready.

[source,bash]
----
# Confirm DataScienceCluster is Ready
oc get dsc default-dsc -o jsonpath='{.status.phase}{"\n"}'

# Check your user project and workbench
oc get notebook -n ai-%USER_NAME%

# List MaaS models / MCP registration
oc get configmap ods-mcp-server-registration -n maas-workshop -o yaml 2>/dev/null | head -20
----

== Features, benefits & cloud configuration

=== Key features

* **DataScienceCluster** (`default-dsc`) — dashboard, workbenches, KServe RawDeployment MaaS proxies.
* Per-user project **`ai-%USER_NAME%`** with **`neuroface-ml-lab`** workbench.
* MaaS playground in `maas-workshop` — shared LLM for all AI modules.

=== Business benefits

* Data scientists and developers share one governed inference layer.
* Notebooks stay on-cluster — no export of factory data to public SaaS.
