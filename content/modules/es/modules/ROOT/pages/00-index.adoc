== Bienvenida

Bienvenido al **Taller Hybrid Mesh AI** — una experiencia dual que refleja cómo los clientes de Red Hat operan nube híbrida en OpenShift. La Parte A (módulos 01–05) es ejecutiva: estrategia, arquitectura ROSA, seguridad a escala, patrones de integración AWS y casos reales. La Parte B (módulos 10–28) es práctica en una flota hub-spoke RHDP con ACM, GitOps, Industrial Edge, OpenShift AI y NeuroFace.

Regístrate en `https://workshop-registration.{hub_domain}` para recibir tu identidad de lab `pass:[%USER_NAME%]`. El Showroom en `https://showroom-showroom.{hub_domain}` incluye terminal `oc` autenticada para seguir la Parte B sin kubeconfig local. Si el scaffolder falla, usa demos compartidos **Plan B** en Developer Hub → System `hybrid-mesh-shared-demos`.
== Catálogo de productos Red Hat en este taller

=== OpenShift Container Platform
* **Advanced Cluster Management (ACM)** — gobernanza de flota, políticas y placement GitOps entre hub y spokes.
* **OpenShift GitOps** — Applications y ApplicationSets de Argo CD sincronizan componentes del taller desde Git.
* **Red Hat Advanced Cluster Security (ACS)** — detección de amenazas y cumplimiento sin mesh ambient en `stackrox`.
* **Kairos** — recomendaciones SmartScalingPolicy para cargas de sensores Industrial Edge.
OpenShift es el plano de control consistente on-prem, edge o ROSA en AWS.

=== Red Hat Application Foundation
* **OpenShift Service Mesh (OSSM 3)** — mesh ambient con ztunnel, Kiali y trazas distribuidas para apps IE.
* **Apache Camel on Kubernetes** — integraciones y pipelines CDC en el demo Industrial Edge.
* **Connectivity Link / Kuadrant** — APIProduct, AuthPolicy y rate limiting en el gateway del hub.
Application Foundation entrega conectividad, integración y gestión de APIs como operadores nativos de OpenShift.

=== Red Hat Advanced Developer Suite
* **Developer Hub (Backstage)** — catálogo, plantillas scaffolder y Topology multicluster.
* **Software Templates** — golden paths para Industrial Edge, Camel, CNV y workspaces OpenShift AI.
* **Red Hat Dev Spaces** — IDE en la nube para Kaoto, Continue AI y desarrollo guiado por plantillas.
Developer Hub convierte OpenShift en plataforma interna de desarrollo con guardrails en cada plantilla.

=== OpenShift AI
* **DataScienceCluster (DSC)** — operador unificado para notebooks, serving y model mesh en el hub.
* **Model-as-a-Service (MaaS)** — endpoint LLM compartido por NeuroFace, Lightspeed y DevSpaces.
* **NeuroFace** — detección objeto/rostro por webcam más chat vía MaaS (no LibreChat).
OpenShift AI mantiene modelos, pipelines e inferencia en el mismo sustrato Kubernetes que tus aplicaciones.

=== OpenShift Virtualization
* **Virtualización nativa de contenedores (CNV)** — cargas VM junto a contenedores vía plantilla demo CNV.
CNV permite lift-and-shift de sistemas legacy de fábrica mientras microservicios nuevos corren en el mismo proyecto OpenShift.

=== Comunidad y terceros
* **Gitea** — org por usuario `ws-pass:[%USER_NAME%]` almacena repos Git generados para Argo CD.
* **MinIO** — almacenamiento objeto para ETL federado Kubecost y artefactos de modelos.
* **Kubecost** — asignaciones FinOps entre primario hub y agentes spoke.
* **Quay** — registro de contenedores referenciado por GitOps y salidas de plantillas.
Estos componentes se integran vía operadores OpenShift y GitOps, como en stacks best-of-breed sobre OCP.

== Panorama

Este taller demuestra cómo los clientes de Red Hat unifican estrategia y operaciones en nube híbrida usando OpenShift como plataforma común. Verás los mismos patrones usados con ROSA en AWS aplicados a un lab hub-spoke RHDP con spokes east y west gestionados por ACM.

La Parte A enmarca el caso de negocio: modernización, seguridad, FinOps e IA sin lock-in. La Parte B te permite desplegar cargas Industrial Edge, observarlas multicluster, asegurar APIs con Kuadrant y ejecutar inferencia con OpenShift AI y NeuroFace — todo como `pass:[%USER_NAME%]` en un slice de namespace aislado.
