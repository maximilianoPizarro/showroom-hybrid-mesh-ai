== Overview

NOTE: **Workshop pacing (~4 hours hands-on)** — Each module includes **Overview-only** steps (demo/visualization, ~5–10 min) and **Hands-on** steps (full lab). Skip Hands-on blocks to run a ~90-minute executive tour.

The **AI Gateway** host (`ai-gateway.%HUB_DOMAIN%`) is separate from **workshop-apis** — both ship in `charts/all/workshop-kuadrant-apis/` but use different Gateway API namespaces and routes. LLM traffic matches path prefix **`/v1`** and proxies to RHDP LiteMaaS; Kuadrant injects the upstream **Bearer** token so workshop users only hold **APIKEY** credentials.

NOTE: The upstream MaaS Bearer belongs in Vault (`secret/workshop/maas`) and should reach the gateway via **ExternalSecret** — see xref:20-vault-external-secrets.adoc[Vault & External Secrets]. RHDP facilitators may use `scripts/apply-maas-secrets.sh` until ESO is wired.

=== Why govern AI inference with an API gateway?

Without a gateway, every factory app needs the raw MaaS URL and shared credentials — no audit trail, no rate limits, no plan tiers. The AI Gateway gives `%USER_NAME%` a Kuadrant key scoped to **free** or **gold** plans: auth at the edge, **TokenRateLimitPolicy** on completion tokens, stable hostname for OT integrations.

*HTTPRoute* — dedicated ai-gateway host (not workshop-apis):

[source,yaml]
----
# charts/all/workshop-kuadrant-apis/templates/ai-routes.yaml
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: ai-maas
  namespace: ai-gateway-system          # dedicated LLM gateway namespace
  labels:
    app: workshop-maas
    kuadrant.io/exposed: "true"
spec:
  parentRefs:
    - name: ai-gateway
      namespace: ai-gateway-system
  hostnames:
    - ai-gateway.%HUB_DOMAIN%          # NOT workshop-apis — separate Route/host
  rules:
    - matches:
        - path: { type: PathPrefix, value: /v1 }
      filters:
        - type: URLRewrite
          urlRewrite:
            hostname: maas-rhdp.apps.maas.redhatworkshops.io
            path: { type: ReplacePrefixMatch, replacePrefixMatch: /v1 }
      backendRefs:
        - group: networking.istio.io
          kind: Hostname
          name: maas-rhdp.apps.maas.redhatworkshops.io
          port: 443
----

*AuthPolicy* — workshop user presents APIKEY; gateway adds upstream MaaS Bearer:

[source,yaml]
----
# charts/all/workshop-kuadrant-apis/templates/policies.yaml (ai-maas excerpt)
apiVersion: kuadrant.io/v1
kind: AuthPolicy
metadata:
  name: ai-maas-auth
  namespace: ai-gateway-system
spec:
  targetRef:
    kind: HTTPRoute
    name: ai-maas
  rules:
    authentication:
      api-key-users:
        credentials:
          authorizationHeader:
            prefix: APIKEY              # client header: Authorization: APIKEY <key>
    response:
      success:
        filters:
          headers:
            authorization:
              plain:
                value: "Bearer <maas-upstream-token>"   # injected — not in Git
----

*TokenRateLimitPolicy* — counts tokens on `/v1/chat/completions`:

[source,yaml]
----
apiVersion: kuadrant.io/v1alpha1
kind: TokenRateLimitPolicy
metadata:
  name: ai-maas-token-limits
  namespace: ai-gateway-system
spec:
  targetRef:
    kind: HTTPRoute
    name: ai-maas
  limits:
    free:
      rates:
        - limit: 50
          window: 1m
      when:
        - predicate: request.path == "/v1/chat/completions"
      counters:
        - expression: auth.identity.userid
----

*APIProduct* — publishes Swagger link in Developer Hub:

[source,yaml]
----
apiVersion: devportal.kuadrant.io/v1alpha1
kind: APIProduct
metadata:
  name: workshop-llm-tokens
  namespace: ai-gateway-system
spec:
  displayName: AI Gateway — MaaS LLM (TokenRateLimit)
  documentation:
    openAPISpecURL: https://raw.githubusercontent.com/openai/openai-openapi/master/openapi.yaml
  targetRef:
    kind: HTTPRoute
    name: ai-maas
----

== Hands-on — terminal and curl

[source,bash]
----
kuadrant-set-key "<your-kuadrant-api-key-from-developer-hub>"
kuadrant-test-ai

# Or manual curl (model granite-3-2-8b-instruct — same as Developer Hub Lightspeed):
export KEY="$KUADRANT_API_KEY"
export MAAS_MODEL="${MAAS_MODEL:-granite-3-2-8b-instruct}"

curl -sk -o /dev/null -w 'LLM without key: %{http_code}\n' \
  -H "Content-Type: application/json" \
  -X POST "https://ai-gateway.%HUB_DOMAIN%/v1/chat/completions" \
  -d "{\"model\":\"${MAAS_MODEL}\",\"messages\":[{\"role\":\"user\",\"content\":\"ping\"}],\"max_tokens\":5}"

curl -sk "https://ai-gateway.%HUB_DOMAIN%/v1/chat/completions" \
  -H "Authorization: APIKEY $KEY" \
  -H "Content-Type: application/json" \
  -d "{\"model\":\"${MAAS_MODEL}\",\"messages\":[{\"role\":\"user\",\"content\":\"Summarize hybrid mesh in one sentence.\"}],\"max_tokens\":50}"

oc get authpolicy,planpolicy,tokenratelimitpolicy -n ai-gateway-system
oc get apiproducts -n ai-gateway-system
----

Compare latency and HTTP **429** (token limit exceeded) with a direct call from **workshop-notebook** in `ai-%USER_NAME%` (module 22).

== Features, benefits & cloud configuration

=== Key features

* Dedicated Gateway **`ai-gateway.%HUB_DOMAIN%`** — LLM isolated from demo APIs on workshop-apis.
* **Kuadrant AuthPolicy** + **TokenRateLimitPolicy** — APIKEY per user, token buckets on completions.
* **Swagger** in Developer Hub via APIProduct OpenAPI URL.

=== Business benefits

* Factory apps get stable external API with auth — no shared MaaS credentials in OT scripts.
* Rate limits protect shared MaaS from runaway loops on the shop floor.
